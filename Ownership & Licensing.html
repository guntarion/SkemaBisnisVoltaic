Struktur Kemitraan dan Kepemilikan Platform Manajemen Inovasi

1. Struktur Kepemilikan dan Lisensi

Platform Inti

  • Pemilik: JTB
  • Jenis: Proprietary
  • Hak: Modifikasi, Distribusi, Komersialisasi

Fitur Khusus Klien

  • Kepemilikan: Bersama (JTB dan PROXSIS)
  • Hak Distribusi: Bersama
  • Komersialisasi: Dapat digunakan untuk klien lain

2. Kepemilikan Komponen Teknis

Front-end (Next.js di Vercel)

  • Kode: Milik JTB
  • Deployment: Dikelola JTB melalui Vercel
  • Kustomisasi: JTB & PROXSIS bersama

Model AI

  • Model Dasar (DeepSeek, Llama 2, Qwen):
    • Lisensi penggunaan: JTB
    • Hak penggunaan: Sesuai ketentuan masing-masing penyedia
  • Model Hasil Fine-tuning:
    • Model terlatih: Milik JTB
    • Proses pelatihan: Milik JTB
    • Kode pelatihan: Milik JTB
    • Data pelatihan: Milik Klien

Komponen Data

  • MongoDB:
    • Data: Milik Klien
    • Skema: Milik JTB
    • Query: Milik JTB
    • Infrastruktur: Milik JTB
  • Database Vektor:
    • Vektor: Milik Klien
    • Proses Embedding: Milik JTB
    • Infrastruktur: Milik JTB
  • Penyimpanan S3:
    • File tersimpan: Milik Klien
    • Struktur penyimpanan: Milik JTB
    • Infrastruktur: Milik JTB
  • Redis Cache:
    • Data cache: Sementara/JTB
    • Logika caching: Milik JTB
    • Infrastruktur: Milik JTB

3. Penanganan Data

Data Klien

  • Kepemilikan: Klien
  • Hak penggunaan: Terbatas untuk penyediaan layanan
  • Retensi: Sesuai perjanjian
  • Penghapusan: Atas permintaan/terminasi kontrak

Aset Turunan

  • Model: JTB dengan lisensi data klien
  • Algoritma: JTB
  • Optimisasi: JTB
Catatan Penting:
  • Semua modifikasi pada fungsi platform inti adalah melalui persetujuan JTB
  • Fitur yang terinspirasi dari klien dapat diintegrasikan ke platform inti
  • Diperlukan dokumentasi yang jelas tentang kepemilikan fitur kustom
  • Data klien tetap menjadi hak milik klien meskipun digunakan untuk pelatihan model
  • Infrastruktur dan proses teknis tetap di bawah kendali JTB
